Question-01: What are the main difference between local storage and session storage?

The Browser Storage or the Client-side Storage works on similar principles to the server-side storage but has different use cases. It consists of JavaScript APIs that allow us to store data on the client (i.e., on the user's machine), and then it could be retrieved when needed. Some of the differences between session storage and local storage are given below:

  • localStorage is similar to sessionStorage, except that while data stored in localStorage has no expiration time, data stored in sessionStorage gets cleared when the page session ends â€” that is, when the page is closed.
  • data stored in either localStorage or sessionStorage is specific to the origin of the page, which is a combination of the protocol, host and the port.
  • Session storage stores data for a specific period of time. This means that SessionStorage data life is for a limited time. As long as the user closes the tab, session storage data also remove from the session storage. On the other hand, when we store data in local storage it stores that data permanently in the browser. Whenever the user returns to that tab he gets back all the data that he stores in localStorage.

    • Question-02: difference between Global scope and block scope?

    "Scope" means which variable is accessible and up to which point it is accessible. In javascript, the whole document is the global scope and all the other functions and variables are contained in this global scope.

    Another is the Local Scope when we declared variables inside the functions are considered to be of the local scope and it is further divided into function scoped and block scope. A variable when declared inside the if or switch conditions or inside for or while loops, are accessible within that particular condition or loop.

    To be consistent the variables declared inside the curly braces are called within block scope. var is called a function scope that is if a variable is declared using the var keyword it will be accessible throughout the function.

    let & const are also called block scope that is they are accessible within that particular block. Here let is used for variables that can be changed as we proceed through the program whereas const is used for a variable that doesn't change till the program ends, that is it remains constant throughout the program.

    Question 03: Discuss how Js Event Loop work?

    The event loop is the secret behind JavaScript's asynchronous programming. JS executes all operations on a single thread, but using a few smart data structures, it gives us the illusion of multi-threading.

    Whenever an async function is called, it is sent to a browser API. These are APIs built into the browser. Based on the command received from the call stack, the API starts its own single-threaded operation.

    The Event Loop has one simple job to monitor the Call Stack and the Callback Queue. If the Call Stack is empty, it will take the first event from the queue and will push it to the Call Stack, which effectively runs it. Such an iteration is called a tick in the Event Loop. Each event is just a function callback.

  • declare var but value not initialized. When we declare a variable but we didn't set any value then it will show an undefined result. Example:
    let first;//undefined
  • function with no return will also show an undefined result. Example:
    function second(a, b) {
    const total = a + b;
    }
    const result = second();//undefined
  • If we declare parameters but will not passed will also output and undefined value. Example:
    function third(a, b, c, d) {
    const total = a + b + c + d;
    console.log(a, b, c, d); }
    third(2, 3);// 2 3 undefined undefined
  • if return has nothing on the right side will also return undefined. Ex:
    function noNegative(a, b) {
    if (a < 0 || b < 0) {
    return;
    }
    return a + b;
    }
    const total=noNegative(2, -5);//undefined
  • Property that doesn't exist on an object will give an undefined value. Ex:
    const obj = { name: "ismail", age: 23 };
    console.log(obj.profession);
    //undefined
  • accessing array element outside of the index range will give undefined. Ex:
    const arr = [12, 3, 4, 5]
    console.log(arr[5]);//undefined
  • if we delete and array element then want to get it.
    const arr = [12, 3, 4, 5]
    delete arr[1];
    console.log(arr);//[ 12, <1 empty item>, 4, 5 ]
  • set a value directly to undefined. If we set a variable value to undefined then it will also return an undefined result. Ex:
    const age = undefined;
    console.log(age);//undefined
